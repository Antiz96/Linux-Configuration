##Dashy
I also installed Dashy on my Raspberry Pi, in order to get an easy access to all my various self hosted services
https://github.com/lissy93/dashy

#Create a dedicated dashy directory
$ sudo mkdir /opt/dashy

#Install and enable Docker
$ sudo pacman -S docker
$ sudo systemctl enable --now docker containerd

#Download my dashy config file
$ sudo curl https://raw.githubusercontent.com/Antiz96/Linux-Customisation/master/Rasp-Server/conf.yml -o /opt/dashy/conf.yml

#Pull and run the container 
--I'm using the "--add-host" docker option to specify the IP of my local service so dashy can check their status (due to its specific environment on my home network, my dashy container cannot access to my local DNS records), but that's optionnal and you probably won't need it. My case's a bit special here--
--I'm using the dashy:arm64v8 image since the Raspberry Pi 4 is an ARM8 board but you can use the dashy:arm37v7 image for ARM7, the dashy:latest image for x86_64 or the dashy:multi-arch which should work on every architecture--
$ sudo docker run -d -p 8080:80 -v /opt/dashy/conf.yml:/app/public/conf.yml --add-host=pihole:192.168.1.1 --add-host=proxmox:192.168.1.100 --add-host=nextcloud:192.168.1.100 --add-host=mabbox.bytel.fr:192.168.1.254 --name dashy --restart=always lissy93/dashy:arm64v8

##Update procedure
$ sudo docker pull lissy93/dashy:arm64v8
$ sudo docker stop dashy
$ sudo docker rm dashy
$ sudo docker run -d -p 8080:80 -v /opt/dashy/conf.yml:/app/public/conf.yml --add-host=pihole:192.168.1.1 --add-host=proxmox:192.168.1.100 --add-host=nextcloud:192.168.1.100 --add-host=mabbox.bytel.fr:192.168.1.254 --name dashy --restart=always lissy93/dashy:arm64v8
